# Artemis

Artemis is a Go template renderer that walks a directory of Go templates,
renders them against a context of build metadata, and writes a `metadata.json`
describing the build. It is designed to be driven by Flox.

## Usage

```
artemis [flags]
```

| Flag              | Default                              | Description                                         |
|-------------------|--------------------------------------|-----------------------------------------------------|
| `--templates-dir` | `./assets/templates`                 | Path to the templates directory                     |
| `--output-dir`    | `$out` if set, else `./build`        | Path to the output directory                        |
| `--initiator`     | env-var lookup (see below)           | Identity of the build initiator                     |
| `--version`       | _(flag)_                             | Print version and exit                              |

### Initiator resolution

`--initiator` falls back through the following environment variables when the
flag is not supplied: `FLOX_BUILD_INITIATOR`, `GITHUB_ACTOR`, `CI_JOB_NAME`,
`USER`. If none are set, the value is `unknown`.

### Quick iteration

```sh
flox activate
go run ./cmd/artemis --templates-dir assets/templates --output-dir build
```

### Flox build

```sh
flox build artemis          # compile the binary
flox build artemis-output   # render templates; result lands in result-artemis-output/
```

## Adding new templates

1. Create a file inside `assets/templates/` with a `.tmpl` extension, e.g.
   `assets/templates/config.yaml.tmpl`. The output file will have the `.tmpl`
   suffix stripped: `config.yaml`.

2. Subdirectories are walked recursively. A file at
   `assets/templates/sub/app.conf.tmpl` renders to `<output-dir>/sub/app.conf`.

3. Files **without** a `.tmpl` extension are copied verbatim, preserving their
   file mode.

### Template context

Every template receives a `TemplateCtx` struct:

| Field        | Type                | Description                                  |
|--------------|---------------------|----------------------------------------------|
| `GitCommit`  | `string`            | Full commit SHA (or env-var fallback)        |
| `GitBranch`  | `string`            | Branch name                                  |
| `GitDirty`   | `bool`              | `true` if the working tree has uncommitted changes |
| `Timestamp`  | `time.Time`         | Build time (UTC)                             |
| `Initiator`  | `string`            | Build initiator identity                     |
| `Version`    | `string`            | Binary version (from ldflags)                |
| `Env`        | `map[string]string` | Allowlisted environment variables            |

### Helper functions

| Function              | Description                                       |
|-----------------------|---------------------------------------------------|
| `upper "text"`        | Convert to uppercase                              |
| `lower "text"`        | Convert to lowercase                              |
| `default "fb" .Val`   | Return `.Val` if non-empty, else `"fb"`           |
| `toYaml .Val`         | Marshal `.Val` to a YAML string                   |
| `toJson .Val`         | Marshal `.Val` to a JSON string                   |
| `env "NAME"`          | Read environment variable `NAME`                  |

Example template:

```
# generated by artemis {{ .Version }} on {{ .Timestamp.Format "2006-01-02" }}
commit: {{ .GitCommit }}
branch: {{ .GitBranch | lower }}
dirty:  {{ .GitDirty }}
```

## Output

After a successful run the output directory contains:

- Rendered files at paths mirroring the template tree (`.tmpl` suffix removed).
- Verbatim copies of any non-`.tmpl` files.
- `metadata.json` — written last so it is **not** included in the source hash.

### `metadata.json` schema

```json
{
  "schemaVersion": 1,
  "tool":   { "name": "artemis", "version": "0.1.0" },
  "source": { "templatesDir": "assets/templates", "hash": "sha256:..." },
  "git":    { "commit": "abcdef...", "branch": "main", "dirty": false },
  "build":  { "timestamp": "2026-04-15T12:34:56Z", "initiator": "alice" }
}
```

`source.hash` is a single aggregate SHA-256 computed over a sorted list of
`(relpath, mode, sha256(content))` tuples for every file in the templates
directory.

## Impurity note

The `[build.artemis-output]` Flox target runs with `sandbox = "off"`. This is
deliberate: the git SHA, branch name, and build initiator are impure inputs
that must flow into `metadata.json` at render time. The binary build
(`[build.artemis]`) is kept as a separate, sandboxed target so the compiled
artifact remains cacheable.
